RELATIONSHIPS
================================================================================

1. INHERITANCE
   Team <|-- CybersecurityTeam
   Team <|-- AITeam
   Team <|-- WebDevTeam
   Team <|-- MobileAppTeam

2. COMPOSITION
   TeamList *-- Team (contains many)
   HackathonModel *-- TeamList (has-a)
   HackathonModel *-- Judge (contains many)
   HackathonModel *-- Category (contains many)
   HackathonModel *-- User (contains many)
   Judge *-- Category (assigned categories)
   Score --> Team (references)
   Score --> Judge (references)

3. ASSOCIATION
   HackathonController --> HackathonModel (uses)
   HackathonController --> HackathonMainView (uses)
   HackathonMainView --> HackathonController (references)
   FileManager ..> TeamList (creates)
   FileManager ..> Team (creates)

4. DEPENDENCY
   HackathonController ..> FileManager (uses static methods)
   TeamList ..> Team (operates on)

THREE-TIER ARCHITECTURE
================================================================================

PRESENTATION TIER:
- HackathonMainView (GUI panels and components)
- HackathonManager (Console interface)

LOGIC TIER:
- HackathonController (MVC Controller)
- HackathonModel (Business logic)
- TeamList (Collection operations)

DATA TIER:
- FileManager (File I/O operations)
- Team classes (Data entities)
- Supporting classes (Category, Judge, Score, etc.)

KEY DESIGN PATTERNS
================================================================================

1. MODEL-VIEW-CONTROLLER (MVC)
   - Separates concerns
   - Model manages data
   - View handles UI
   - Controller coordinates

2. INHERITANCE
   - Team base class
   - Category-specific subclasses
   - Polymorphic behavior

3. SINGLETON (Implied)
   - FileManager uses static methods
   - Central point for file operations

4. FACADE
   - TeamList provides simplified interface to team operations
   - HackathonController facades complex interactions

5. FACTORY METHOD
   - FileManager.parseTeamLine() creates appropriate team type
   - Controller.createTeam() based on category

SCALABILITY CONSIDERATIONS
================================================================================

1. New Team Categories:
   - Create new subclass extending Team
   - Implement getOverallScore()
   - Update FileManager parsing logic

2. New Features:
   - Add methods to Model
   - Create new View panels
   - Add Controller handlers

3. Database Integration:
   - Replace FileManager with DatabaseManager
   - Implement same interface
   - No changes to other classes needed

4. Web Interface:
   - Keep Model and Controller
   - Replace View with web framework
   - RESTful API possible
